# Coding Platform

## Phase 1 Objective

Bring the coding product to a production MVP baseline.

## Phase 1 User Capabilities

- browse problems
- open the coding workspace
- run code
- run code with custom input
- submit code
- receive verdicts

## Phase 1 Admin Capabilities

- create and edit problems
- manage testcases
- manage language and template content

## Current Phase 1 Focus

- stop trusting frontend-supplied user identity
- fix testcase persistence
- support real custom input execution
- strengthen submission ownership handling
- improve coding workspace behavior

## Remaining Phase 1 Verification Areas

- security config consistency
- submission history and detail completeness
- broader backend tests
- admin problem authoring completeness
